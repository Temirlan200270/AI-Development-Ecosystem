# Инструкция по установке зависимостей

## Проблема
Модуль `diff_match_patch` не установлен, что вызывает ошибку при запуске.

## Решение

### Вариант 1: Установка всех зависимостей (рекомендуется)
```powershell
cd "C:\AI Development Ecosystem"
pip install -r requirements.txt
```

Или если pip не найден:
```powershell
python -m pip install -r requirements.txt
```

### Вариант 2: Установка только недостающего модуля
```powershell
pip install diff-match-patch
```

Или:
```powershell
python -m pip install diff-match-patch
```

### Вариант 3: Установка проекта как пакета
```powershell
pip install -e .
```

Это установит все зависимости из `pyproject.toml`.

## Правильный способ запуска

### ❌ Неправильно:
```powershell
python run test_mercury.py
```

### ✅ Правильно:

**Вариант 1: Через модуль Python**
```powershell
python -m temir.main run "ваш запрос пользователя"
```

**Вариант 2: Если установлен как пакет**
```powershell
temir run "ваш запрос пользователя"
```

**Вариант 3: Прямой запуск main.py**
```powershell
python temir\main.py run "ваш запрос пользователя"
```

## Примеры команд

```powershell
# Создать простое приложение
python -m temir.main run "Создай простое приложение TODO на Python"

# С флагами
python -m temir.main run "Создай REST API" --output-dir ./my_project --no-sandbox

# Просмотр конфигурации
python -m temir.main config-cmd
```

## Проверка установки

После установки проверьте:
```powershell
python -c "import diff_match_patch; print('OK')"
```

Если выводит "OK", значит модуль установлен правильно.
